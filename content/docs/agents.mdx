## The Agent Runner

Launch any CLI coding agent from the top bar. Claude Code, Codex, Gemini, Grok and OpenCode come preconfigured; anything else drops into the registry.

Every running agent shows a **status pill**: `working` (spinner), `done` (check), `blocked` / `waiting` / `permission` (red), `idle` (gray). Click a pill to jump to that agent's tab.

Status comes from two sources:

1. **Hook pushes** — a listener on `127.0.0.1:<port>/v1/hook` accepts real-time events from agent hook scripts (per-session bearer tokens).
2. **Output-silence detection** — a fallback for agents without hooks.

## The registry

Agents are defined in `~/.config/xnaut/agents.toml`. Each entry names the binary, how to inject a prompt (five strategies cover every CLI's quirks), and how to detect state. Add your own:

```toml
[agents.myagent]
command = "myagent"
prompt_mode = "argv"   # argv | stdin | flag | interactive | file
```

## One worktree per agent

Parallel agents editing the same checkout is how merge conflicts are born. The **Worktree Manager** (top bar) creates isolated git worktrees with a recipe that keeps `git push` and `git status` behaving (`--no-track` + `push.autoSetupRemote=true`), and can launch any registered agent directly inside the new tree.

## Reviewing agent work

Open the **Diff Viewer** on any worktree for a side-by-side `git diff HEAD`. Agents leave inline annotations — summary, rationale, confidence — anchored to exact hunks via `<worktree>/.xnaut/notes.json`, and the pane updates within ~100 ms of changes. An HTTP broker at `127.0.0.1:<port>/v1/notes` gives external agents the full comment vocabulary.
